Major Bug Fixed: 2FA / OTP Input and Extension Memory

Date: June 25, 2026

1. Typing Follows the Cursor (Focus-Aware Input)
- The Problem: The automation stubbornly tried to force all 6 numbers of a 2FA code into the first input box. It ignored the fact that OTP components automatically advance the focus to the next box after a character is typed.
- The Technical Fix: 
  - Playwright: Swapped `locator.pressSequentially()` for `page.keyboard.type()`. `page.keyboard.type()` dispatches keystrokes to `document.activeElement`, allowing it to naturally follow focus as the OTP component auto-advances.
  - Extension Runner: Rewrote the `Type` loop in `content_runner.js`. Instead of firing events statically on the initial `target`, it re-evaluates `document.activeElement` on every character keystroke. This mirrors real browser behavior for focus-shifting fields.

2. Variable Storage & Memory
- The Problem: The Chrome Extension previously forgot variables generated by Javascript steps (like the 2FA code). It had no memory to pass the result into subsequent steps.
- The Technical Fix: Added a state dictionary (`sfqaEnv`) to the extension playback loop (`background.js`). It now correctly captures the returned Javascript OTP result and interpolates variable placeholders (e.g., `{{2FA_Code}}`) dynamically before executing the Input step.
